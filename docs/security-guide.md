# Security Guide

## 1. Аутентификация и авторизация

- JWT access token для защищенных API.
- Проверка прав администратора через `users.is_admin`.
- Поддержка pre-2FA токена для безопасного шага входа с 2FA.

## 2. OTP и 2FA

- Email OTP и SMS OTP с TTL и ограничениями по частоте.
- TOTP 2FA на базе `otplib`.
- Backup-коды для восстановления доступа.

## 3. Защита API

- `express-rate-limit` для ограничения злоупотреблений.
- CORS с белым списком origin.
- Валидация входных данных на уровне endpoint-ов.

## 4. Секреты и переменные окружения

- `JWT_SECRET`, `WEBPAY_SECRET_KEY`, `HF_API_TOKEN`, SMTP/SMS ключи хранятся только в env.
- Секреты не должны коммититься в репозиторий.

## 5. Безопасность оплаты

- Подпись WebPay-параметров при инициации платежа.
- Проверка подписи в callback `/api/webpay-notify`.
- Идемпотентная обработка повторных уведомлений.
