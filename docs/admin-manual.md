# Руководство администратора TudaSuda

## 1. Назначение

Документ описывает работу администратора в панели управления TudaSuda.

[СКРИНШОТ: вход в админ-панель]

## 2. Доступ в админ-панель

1. Войти в систему под учетной записью с `is_admin = true`.
2. Перейти в раздел `/admin`.

Если прав недостаточно, API вернет `403 Admin access required`.

## 3. Дашборд и аналитика

Раздел дашборда предоставляет:
- общее число пользователей;
- число билетов и заказов;
- выручку по оплаченным билетам;
- графики по статусам и динамике.

[СКРИНШОТ: admin dashboard]

## 4. Управление блогом (CRUD)

### 4.1 Создание статьи
1. Открыть раздел блога.
2. Нажать `Создать`.
3. Заполнить slug, заголовок, контент.
4. Сохранить.

### 4.2 Редактирование
1. Открыть статью в списке.
2. Изменить данные.
3. Сохранить.

### 4.3 Удаление
1. Выбрать статью.
2. Нажать `Удалить`.
3. Подтвердить действие.

[СКРИНШОТ: управление блогом]

## 5. Управление маршрутами (CRUD route pages)

Админ может:
- создать страницу маршрута;
- отредактировать существующую;
- удалить неактуальную;
- управлять SEO-полями и контентом.

[СКРИНШОТ: управление маршрутами]

## 6. Работа с заказами и пользователями

Типовые операции:
- просмотр данных заказа;
- проверка статусов оплаты;
- анализ активности пользователя по журналам/статистике.

[СКРИНШОТ: раздел заказов]

## 7. Типовой регламент администратора

1. Проверить dashboard после входа.
2. Обработать новые публикации и маршрутные страницы.
3. Контролировать ошибки оплаты/поддержки.
4. Проверить корректность данных в публичной части.

## 8. Безопасность доступа

- Использовать сложный пароль и 2FA.
- Не передавать токены третьим лицам.
- Работать в защищенном контуре (HTTPS, доверенное устройство).
