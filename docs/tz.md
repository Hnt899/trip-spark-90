# Техническое задание (краткая версия)

## 1. Основание для разработки

Разработка выполняется в рамках договора с заказчиком ИП Пилипенко Д.А. и дипломного проекта по специальности 09.02.07.

## 2. Назначение системы

Создание веб-сервиса TudaSuda для поиска, бронирования и оплаты билетов с административной панелью и модулем поддержки.

## 3. Требования к функциональности

### Пользовательская часть
- регистрация/авторизация (пароль, OTP);
- поиск маршрутов и выбор мест;
- оформление и оплата заказа;
- личный кабинет, история билетов;
- возврат/обмен;
- чат-помощник.

### Административная часть
- авторизованный доступ администраторов;
- dashboard и аналитика;
- CRUD блога;
- CRUD маршрутных страниц;
- контроль контента.

## 4. Нефункциональные требования

- безопасность: JWT, OTP, 2FA, rate-limit, CORS, хранение секретов в env;
- надежность: health-check endpoint, журналирование ошибок;
- производительность: приемлемое время ответа API для пользовательских операций.

## 5. Требования к ПО и инфраструктуре

- frontend: React + TypeScript + Vite;
- backend: Node.js + Express;
- БД: PostgreSQL;
- deployment: Vercel + Railway.

## 6. Этапы работ

1. Анализ требований.
2. Проектирование архитектуры и БД.
3. Реализация frontend/backend.
4. Интеграции (оплата, OTP, чат-помощник).
5. Тестирование.
6. Подготовка документации и передача заказчику.

## 7. Приемка

Критерии приемки:
- реализован заявленный функционал;
- работоспособность основных пользовательских и админ-сценариев;
- наличие комплектной документации.
