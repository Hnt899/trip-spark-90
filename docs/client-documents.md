# Документация для передачи заказчику

## 1. Комплект документов

После завершения работ заказчику (ИП Пилипенко Д.А.) передаются:

1. Техническое задание (`docs/tz.md`).
2. Руководство пользователя (`docs/user-manual.md`).
3. Руководство администратора (`docs/admin-manual.md`).
4. API-документация (`docs/api-reference.md`).
5. Документация по БД (`docs/database.md` + `postgres/schema.sql`).
6. Инструкция по развёртыванию (`DEPLOYMENT.md`).
7. Документация по тестированию (`TESTING.md`).
8. Руководство по чат-помощнику (`docs/support-guide.md`).

## 2. Сопроводительные материалы

- акты выполненных работ (по регламенту заказчика);
- инструкции по эксплуатации;
- архив исходного кода и миграций.

## 3. Результат передачи

Заказчик получает комплект, достаточный для:
- запуска продукта;
- эксплуатации пользовательской и админской частей;
- базовой технической поддержки и развития проекта.
