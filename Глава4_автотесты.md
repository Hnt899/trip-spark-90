# 4.X Автоматизированное тестирование

В рамках разработки проекта TudaSuda в дополнение к ручным проверкам внедрён контур автоматизированного тестирования, охватывающий серверную логику, API-эндпоинты, компонентный frontend-уровень и сквозные пользовательские сценарии.

Для backend-части применён стек **Vitest + Supertest**. Это позволило тестировать критичные endpoint’ы (`auth`, `otp`, `2fa`, `tickets`, `webpay`, `admin`) в изолированном режиме через mock-слой базы данных (`pg`), без необходимости поднимать реальный экземпляр PostgreSQL в каждом прогоне. Для frontend-части использованы **Vitest + Testing Library + jsdom**, что дало возможность проверять рендер ключевых компонентов и базовые пользовательские действия. Для сквозного уровня подключён **Playwright** с тестовым сценарием покупки билета и замоком ответа WebPay.

В результате были внедрены следующие уровни автотестов:

- unit-тесты backend (хелперы, JWT, middleware безопасности);
- API-тесты backend (позитивные и негативные сценарии по критичным маршрутам);
- компонентные тесты frontend;
- e2e-тесты пользовательского потока покупки.

Суммарная статистика автотестов:

| Категория | Количество |
|---|---:|
| Unit + API (backend) | 45 |
| Компонентные (frontend) | 5 |
| E2E (Playwright) | 1 |
| **Итого** | **51** |

Показатели покрытия (отчёт Vitest Coverage):

| Область | Покрытие строк |
|---|---:|
| Backend бизнес-логика и API | 87% |
| Frontend (ключевые компоненты) | 65% |

Все тесты включены в CI-процесс и запускаются автоматически при `push`/`pull_request` в ветку `main` (lint → test → build), что повышает надёжность релизного цикла и снижает риск регрессий.
