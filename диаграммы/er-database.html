Все таблицы с полями updated_at используют триггер update_updated_at_column(). Расширение БД: pgcrypto (UUID и крипто-утилиты для функций 2FA и сертификатов).

Схема CI/CD pipeline

Push в ветку main (или ручной запуск) → обновление на VPS: nginx + Node.js + PostgreSQL (отдельная страница для экспорта cicd.png)

GitHub
Репозиторий проекта · ветка main
push / ручной деплой → git pull и скрипт на сервере
↓ trigger
VPS (Ubuntu LTS)
nginx: npx vite build → каталог dist/ · статика SPA
reverse proxy: /api/* и /uploads/* → процесс Node.js (Express)
старт: copy-seed-images → migrate.js → server/index.js (systemd или PM2)
PostgreSQL на том же сервере · DATABASE_URL → Express / pg

Пользователь открывает сайт на своём домене; nginx отдаёт фронт и проксирует /api и /uploads на backend. Обязательный GitHub Actions pipeline не используется — сценарий деплоя настраивается на VPS при пуше в main (см. DEPLOYMENT.md).

Для файла cicd.png: откройте диаграммы/cicd.html в браузере и сделайте скриншот только блока схемы или экспорт из инструментов браузера.
