Схема CI/CD pipeline

GitHub → VPS (Ubuntu LTS): nginx + Node.js (Express) + PostgreSQL

GitHub
Репозиторий · ветка main · push или ручной запуск → обновление кода на сервере (git pull и скрипт деплоя)
↓ trigger
VPS (Ubuntu LTS)
nginx: статика SPA из dist/ после npx vite build
reverse proxy: /api/*, /uploads/* → процесс Node.js (Express)
старт приложения: copy-seed-images → migrate.js → server/index.js (systemd или PM2)
PostgreSQL на том же сервере · DATABASE_URL → Express / pg

Пользователь открывает сайт на своём домене; nginx отдаёт фронт и проксирует /api и /uploads на backend. Обязательный GitHub Actions pipeline не задан — деплой по main настраивается на сервере (cron, webhook или ручной сценарий, см. DEPLOYMENT.md).
